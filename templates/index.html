Menu

Tome una plantilla libre llamada MeetMe la cual se puede usar libremente siempre y cuando no se elimine el footer que contiene informacion sobre el autor. Tome la parte del menu superior y cree un template base para poder heredarlo en los siguientes templates que iba a usar, de esta forma es mucho mas facil crear menus. Cabe aclarar que todo lo relacionado al frontend esta basado en el template, excepto en algunos casos como por ejemplo, al desabilitar un boton este se veia igual, por lo que retire su clase y se ve como un simple boton de HTML.

El menu esta compuesto por cada uno de los requerimientos del sistema, y en el datatable de las peliculas se puede seleccionar la accion de editar o eliminar. Cabe aclarar que el menu que un usuario sin loguear ve es diferente al de uno ya logueado.

Por cuestiones de tiempo no pude agregar una vista para ver las peliculas favoritas del usuario, el menu de recomendaciones simplemente recomienda de forma aleatoria las peliculas que NO estan en la lista de favoritos (es un query en sql muy sencillo), por cuestiones de tiempo no pude hacerlo personalizado, queria recomendar con respecto al genero y director. Para agregar una pelicula a la lista de favoritos es necesario dar click a la imagen, se recomiendan 5 peliculas a la vez.


Cuestiones tecnicas

La creacion de usuarios y peliculas se realizan a traves de Forms, no los utilice en el frontend porque me parecen muy poco esteticos y por ello preferi hacerlos a partir del template, pero la logica al momento de guardar se realiza con los forms. Queria hacer algo similar para el CRUD de las peliculas, pero al momento de investigar el Django Rest framework me di cuenta que el CRUD se podia hacer a traves de este, me emocione porque personalmente solo lo utilizo para obtener informacion en formato JSON ya sea para trabajar un datatable (como en en listado de las peliculas) o para otros objetivos, pero nunca un CRUD con el, por lo que descarte mi idea original de trabajar con forms y comence a trabajar con el Django Rest, a traves del protocolo Put se edita la pelicula y con el protocolo Delete se elimina del sistema. La imagen y el genero de la pelicula no se pueden edita, tampoco existe un template para la creacion de los generos por lo que hay 2 formas de crearlos, una a traves del ORM y otra con el Django Rest, deje el metodo y el serializador para ser creados accediendo a la url Generos, estos pueden ser creados sin necesidad de estar logueado en el sistema, caso contrario del CRUD de las peliculas.

Para comodidad en ver el aplicativo, lo desplegue en un servidor Heroku, en donde ya existe un usuario y algunas peliculas para poder probar el CRUD, para este despliegue tuve que cambiar de SQLite a Postgresql debido a un problema con Heroku, ademas es posible que presente problemas con las imagenes puesto que estas se guardan en la carpeta MEDIA y Heroku despues de cierto tiempo suele borrarlas (debido a que es un servidor gratis de prueba), pero esto no deberia suceder al probar el proyecto de forma local.

Lamentablemente no poseo los conocimientos para la parte de testing, por lo que el aplication no tiene testing de ningun tipo y su covertura es del 0%. Actualmente estoy haciendo un curso de Jenkins para aprender CI/CD, por lo que al final de este curso espero aprender tambien testing, lo cual es fundamental para mi vida profesional y no quisiera perder otra oferta laboral por la falta de esta.
